# Preferences — Defaults

These files define the default stack, conventions, and constraints loaded for every project.

| File | Owns |
|------|------|
| `tech-stack.md` | Framework choices, state management rules, TypeScript conventions, layer boundary rule |
| `programming-style.md` | Architecture principles, auth pattern, error handling, progressive building techniques |
| `testing.md` | Test types, tools, locations, coverage thresholds, quality rules |
| `libraries-and-mcps.md` | Approved dependencies, MCPs, rules for adding new ones |
| `setup-steps.md` | One-time project scaffolding sequence |
| `user-profile.md` | Communication style, YOLO mode default |
