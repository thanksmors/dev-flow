# Frontend Distill

Remove unnecessary complexity from designs, revealing the essential elements.

## MANDATORY PREPARATION

Invoke the **frontend-design skill**:
- Read `skills/frontend-design/SKILL.md`
- Read `skills/frontend-design/reference/ux-writing.md`

## Assess Current State

Identify complexity sources:
- **Too many elements**: Competing buttons, redundant information
- **Excessive variation**: Too many colors, fonts, sizes without purpose
- **Information overload**: Everything visible at once
- **Visual noise**: Unnecessary borders, shadows, decorations
- **Confusing hierarchy**: Unclear what matters most

Find the essence:
- What's the primary user goal? (ONE thing)
- What's actually necessary vs nice-to-have?

## Plan Simplification

- **Core purpose**: What's the ONE thing this should accomplish?
- **Essential elements**: What's truly necessary?
- **Progressive disclosure**: What can be hidden until needed?

## Simplify the Design

### Information Architecture
- Remove secondary actions, redundant information
- Hide complexity behind accordions, modals, step-through flows
- ONE primary action, few secondary actions, everything else tertiary or hidden

### Visual Simplification
- Reduce color palette to 1-2 colors plus neutrals
- Limit typography to one font family, 3-4 sizes, 2-3 weights
- Remove decorations that don't serve hierarchy
- **Never nest cards inside cards**

### Content Simplification
- Cut every sentence in half
- Active voice
- Plain language
- Short paragraphs, bullet points, clear headings

## Verify Simplification

- Faster task completion?
- Reduced cognitive load?
- Still complete?
- Clearer hierarchy?

**NEVER**:
- Remove necessary functionality
- Sacrifice accessibility
- Make things so simple they're unclear
